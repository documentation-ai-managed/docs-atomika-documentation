const createProject = async (name, description) => {
  const response = await fetch('/api/projects', {

<Expandable title="What HTTP status codes does the API use?" default-open="false">
  We use standard HTTP status codes: 200 for success, 201 for created resources, 400 for bad requests, 401 for unauthorized access, 404 for not found resources, and 500 for server errors.
</Expandable>

## JSON Responses

All API responses are returned in JSON format with a consistent structure. Successful responses include the requested data, while error responses provide detailed error information.

<ExpandableGroup>
  <Expandable title="Success Response Structure" default-open="false">
    Successful responses include a `data` field with the requested resource, and optional `meta` information like pagination details.
  </Expandable>
  <Expandable title="Error Response Structure" default-open="false">
    Error responses include an `error` object with `code`, `message`, and optional `details` for debugging.
  </Expandable>
</ExpandableGroup>

<Callout kind="tip">
  Use the `Accept` header to specify JSON format explicitly: `Accept: application/json`.
</Callout>

## Authentication

Secure your API calls with authentication. We support API key authentication and OAuth 2.0 for more advanced use cases.

<Steps>
  <Step title="Generate API Key" icon="key">
    Create an API key from your account settings.
  </Step>
  <Step title="Include in Requests" icon="send">
    Add the API key to the Authorization header.
  </Step>
  <Step title="Rotate Keys" icon="refresh-cw">
    Regularly rotate your API keys for security.
  </Step>
</Steps>

<CodeGroup tabs="API Key,OAuth">
  ````bash
  # API Key authentication
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       https://api.atomika.com/v1/docs
  ````
  ````javascript
  // OAuth flow example
const getAccessToken = async (code) => {
  const response = await fetch('/oauth/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      grant_type: 'authorization_code',
      code,
      client_id: CLIENT_ID,
      client_secret: CLIENT_SECRET
    })
  });
  return response.json();
};
  ````
</CodeGroup>

<Expandable title="How do I set up OAuth?" default-open="false">
  Register your application in the developer console to get client credentials. Implement the OAuth flow in your application to obtain access tokens.
</Expandable>
